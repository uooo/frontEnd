# 基本使用

静态资源打包工具成一个bundle

由于浏览器无法兼容es6的模块化，模块化开发的问题，因此我们要使用打包工具，集成到一个包中，用一个js文件管理整个页面

Uncaught SyntaxError: Cannot use import statement outside a module

webpack-cli:webpack的指令

初始化npm init -y 生成package.json

安装webpack，npm i webpack webpack-cli -D

打包操作：npx webpack 打包入口地址 --mode=devlopment | production

npx将 node_modules 添加进环境变量

--mode=devlopment

注意使用npx命令： webpack-cli是命令包

  • npx侧重于执行命令的，执行某个模块命令。虽然会自动安装模块，但是重在执行某个命令。
  • npm侧重于安装或者卸载某个模块的。重在安装，并不具备执行某个模块的功能。

宏观概览webpack工作逻辑

  • 一个主js文件main.js，需要引入什么js相关文件就import其中
  • 一个webpack.config,js的webpack配置文件，修改先关配置项入口，输出位置，需要用的加载器（比如loader），插件，模式什么的。
  • npm webpack 启动webpack开始打包静态文件生成静态js文件，使用一个html引入该js测试。

# 如何处理样式资源

本章节我们学习使用 Webpack 如何处理 Css、Less、Sass、Scss、Styl 样式资源，在配置文件webpack.config.js中，用module配置项添加规则rules中添加loader

loader中只可以使用一个loader，而use可以使用多个！！

用loader rules中的use 数组里面 Loader 执行顺序是从右到左，否则报错

loader 从右到左（或从下到上）地取值(evaluate)/执行(execute)。 // use 数组里面 Loader 执行顺序是从右到左

逻辑就是，安装相应的loader，在webpack配置项中添加rules，在test中检测文件后缀，use中添加使用的loader，loader是有使用规则的从后往前的。

逻辑：loader生成css文件，css文件loader成style语句嵌入到html文件中，最后用webpack生成静态使用可以直接使用的文件。

# 如何处理图片资源

不用任何配置，webpack5已经把功能集成好了。

优化：小于10kb的图片就不用发起请求，用base64的方式显示小图片，虽然会大一点，但是可以减少发起请求的数量。

将小于10kb的小图片转为base64，减少网页请求数量

添加到module中的rules数组中

 {
        test: /\.(png|jpe?g|gif|webp)$/,
        type: "asset",
        parser: {
          dataUrlCondition: {
            maxSize: 10 * 1024 // 小于10kb的图片会被base64处理
          }
        },
     	generator: {
                    // 将图片文件输出到 static/imgs 目录中
                    // 将图片文件命名 [hash:8][ext][query]
                    // [hash:8]: hash值取8位
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: "static/imgs/[hash:8][ext][query]",
                  },
 },
